# v0.1.0 Release Notes

## What Works

- Local repository ingestion, source filtering, license-aware citations, and secret redaction.
- Python and TypeScript/JavaScript extraction for symbols, imports, routes, package manifests, docs, tests, and intra-file Python calls.
- Cited context-pack retrieval through CLI, HTTP API, optional FastAPI app factory, and MCP-compatible stdio.
- Reflexion feedback, staged knowledge promotion/rejection, SQLite backup/restore, doctor checks, and local benchmark reporting.

## Tested

- `python3 -m compileall repo_wiki`
- `python3 -m unittest discover -s tests -v`
- `python3 -m ruff check .`
- CLI, HTTP, MCP-compatible stdio, benchmark report, and bundled GraphRAG validation are recorded in `progress.md`.

## Known Limitations

- The real report covers local MVP validation. Do not claim the planned 20-repository MVP benchmark or portfolio-scale 100+ repository numbers until those commands are run against a populated index.
- TypeScript extraction is scanner-based. Add compiler-backed extraction only when judged failures prove it is the bottleneck.
- The MCP path is a thin JSON-RPC stdio adapter. Official SDK adoption should wrap the same core service calls.

## Release recommendation

Ready for GitHub publication as a polished local MVP, with the benchmark-scope limitations above stated plainly.
